üéì ALGORITHM SELECTION RECOMMENDATION FOR CONNECT4
====================================================================================================

WHY WE CHOSE MINIMAX + ALPHA-BETA PRUNING:

1. PROBLEM CHARACTERISTICS:
   - Connect4 is a two-player, zero-sum, perfect information game
   - Adversarial search is required (opponent tries to minimize our score)
   - Game tree is finite but large (branching factor ‚âà 7, depth ‚âà 42)

2. WHY NOT UNINFORMED SEARCH:
   ‚ùå BFS/DFS: Don't model adversarial nature, high memory usage
   ‚ùå UCS: No consideration of opponent's best response
   ‚ùå These are for single-agent pathfinding, not game playing

3. WHY MINIMAX:
   ‚úÖ Designed for two-player games
   ‚úÖ Guarantees optimal play (if fully expanded)
   ‚úÖ Complete and optimal

4. WHY ALPHA-BETA PRUNING:
   ‚úÖ Same result as Minimax, but faster
   ‚úÖ Empirical improvement: ~75.9% fewer nodes
   ‚úÖ Allows deeper search in same time
   ‚úÖ No loss in optimality

5. WHY HEURISTIC EVALUATION:
   ‚úÖ Can't search to end of game (too deep)
   ‚úÖ Heuristic estimates position value
   ‚úÖ Considers: center control, 3-in-a-rows, threats
   ‚úÖ Makes AI play intelligently at limited depth

6. INFORMED vs UNINFORMED:
   - A* is 'informed' (uses heuristic for pathfinding)
   - Alpha-Beta uses heuristic BUT for position evaluation, not search guidance
   - Classification: Minimax is adversarial search (different category)
   - We use heuristics because full tree search is impractical
